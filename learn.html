Hello World

Your first script!

Now that your work enviroment is set up, lets make your first script! What will it do? Well it will be your traditional Hello World program!

Don't freak out! Its actually extremely simple! First, create a file in the scripts folder named hello_world.js. In this file put the following in the file: print("Hello, world"). When you start the server, you will see the log entries displayed below. 

2012-10-01 04:38:17 [INFO] [bScript] Found 1 scripts in script folder
2012-10-01 04:38:17 [INFO] [bScript] Loading script 'hello_world.js'
2012-10-01 04:38:17 [INFO] [bScript] [hello_world.js] Hello, world
As you can see, it loaded the script and then printed out Hello, world. Congratulations, you wrote your first script!

Events

Listen to the server

One of the biggest features of bScript is the fact that it will scan Bukkit and any plugins for Events. This means that no events are hardcoded into bScript meaning that bScript doesn't have to update to new events! It also means that plugins' custom events do not have to be added to bScript. Executing writeevents in the console will get you a list of all of the events. The list will be in plugins/bScript/events.txt and will entries similar to the following.

Simple Name: PlayerInteractEvent

For each event you can use any of the names listed. Lets write a simple script that welcomes players to the server. Here is the code: 
print("Welcome script loaded!");
events.on("PlayerJoinEvent", function(event) {
  event.setJoinMessage("Welcome to the server!");
})
When you log in, you should see a message saying Welcome to the server!

Commands

Interact with the server

With bScript adding commands so simple! Lets add a command to send a message to another player on the server.
Heads up! Two things are passed to your command function, the source and the context. The following methods are available for source: isPlayer(), getPlayer(), isConsole(), sendMessage(String message). The following methods are available for the context: getCommand() and getArgs().
Here is the source for the message command: 

commands.register("msg", function(source, context) {
  if (!source.isConsole()) {
    if (context.getArgs().length < 2) { // If there were two or more args
      source.getPlayer().sendMessage("Correct usage is /msg [player] your message");
    } else {
      var args = context.getArgs();
      // get the player defined in the args
      var player = Bukkit.getPlayer(args[0]);
      if (player != null && player.isOnline()) {
        var msg = "";
        for (var i = 1; i < args.length; i++) {
          msg += args[i] + " ";
        }
        player.sendMessage(source.getPlayer().getName() + " says: " + msg);
      } else {
        source.getPlayer().sendMessage("Recipient is not online!");
      }
    }
  }
})

Colors

Utility functions for colors

There are several functions that you can use to manipulate colors in strings. The available colors are: BLACK, DARK_BLUE, DARK_GREEN, DARK_AQUA, DARK_RED, DARK_PURPLE, GOLD, GRAY, DARK_GRAY, BLUE, GREEN, AQUA, RED, LIGHT_PURPLE, YELLOW, WHITE. There are also a few formatting styles, such as MAGIC, BOLD, STRIKETHROUGH, UNDERLINE, and ITALIC. The RESET format/color will reset all color and format.
The first function is the color() function. It returns the color or format put in the function. Example: color("RED")
The second function is the format function. It will return a string with all the color's and formats inserted. To request a color or format, do {{COLOR OR FORMAT}}. Example: format("This word is {{RED}}red{{RESET}} this is {{UNDERLINE}}underlined{{RESET}}")
You can remove all colors in format by simply calling the function stripColors() on a string. Example: stripColors(myColoredString).

Tasks

Scheduling tasks

One neat feature of Bukkit is the ability to schedule tasks that can be set to repeat or set to run at a later time. bScript has api to allow you schedule tasks. First you need to build a Task. The Tasks class has several methods that you need to know. The first one is setDelay(TimeBuilder delay), it allows you to set the delay before running the task. The second is setRepeatDelay(TimeBuilder repeatDelay), it allows you to set a function to repeat at the delay that you define in the TimeBuilder. The third is setAsync(boolean async), (Warning this should only be used if you know what an async task is, its for experienced users only) it allows you to set whether or not your task will be run asynchronously.
By now you are probably wondering what the TimeBuilder above is and how you can use it. It allows you to set the amount time for each delay or repeat delay. The functions provided are: addTicks(), addSeconds(), addMinutes(), and addHours().
Remember A task with a repeating delay will be executed immediately unless you also set a delay on it.
Lets write an example that schedules a repeating notice to the server. 

var task = new Task();
task.setDelay(new TimeBuilder().addSeconds(30));
task.setRepeatDelay(new TimeBuilder().addMinutes(1).addSeconds(15));

tasks.schedule(task, function() {
  print("Running the task!");
  Bukkit.broadcastMessage("Don't forget to show your support by donating to alta189! THANKS!");
})
